## 🎓 Обучение по шагам

### Шаг 1: Установка

1. Прочитайте [README.md](README.md#-установка)
2. Установите плагин на сервер
3. Запустите сервер для генерации конфигов

### Шаг 2: Базовая настройка

1. Откройте [CONFIGURATION.md](CONFIGURATION.md#главный-конфиг-configyml)
2. Настройте `config.yml` (включите нужные модули)
3. При необходимости настройте базу данных

### Шаг 3: Настройка модулей

1. Изучите [CONFIGURATION.md](CONFIGURATION.md#модули)
2. Настройте каждый включённый модуль в `modules/*.yml`
3. Отредактируйте сообщения в `lang/ru.yml`

### Шаг 4: Выдача прав

1. Откройте [COMMANDS.md](COMMANDS.md)
2. Выдайте права игрокам через LuckPerms или другой плагин прав

### Шаг 5: Тестирование

1. Проверьте работу команд
2. При проблемах обратитесь к [FAQ.md](FAQ.md#проблемы-и-ошибки)

### Шаг 6: Интеграция (для разработчиков)

1. Изучите [DEVELOPER_GUIDE.md](DEVELOPER_GUIDE.md)
2. Подключитесь к API
3. Используйте события и сервисы

## 📝 Глоссарий

| Термин | Значение |
|--------|----------|
| **Модуль** | Отдельная функциональная часть плагина (Warps, Homes, Chat, etc.) |
| **GUI** | Графический интерфейс (меню с предметами) |
| **Кулдаун** | Задержка между использованиями команды |
| **Плейсхолдер** | Переменная в тексте (например, `{player}`, `{balance}`) |
| **MiniMessage** | Форматирование текста с цветами и стилями |
| **YAML** | Формат файлов конфигурации (.yml) |
| **MySQL** | Система управления базами данных |
| **Redis** | Система кэширования для кластеров |
| **API** | Интерфейс для взаимодействия с другими плагинами |
| **EventBus** | Система событий плагина |
| **Profile** | Профиль игрока с данными (дома, кулдауны, метаданные) |
