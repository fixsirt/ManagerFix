## 📥 Установка и обновление

### ❓ Как установить плагин?

**Ответ:**
1. Скачайте файл `ManagerFix-1.0.0.jar`
2. Поместите его в папку `plugins/` на вашем сервере
3. Перезапустите сервер
4. Настройте файлы в `plugins/ManagerFix/`

### ❓ Нужно ли удалять другие плагины перед установкой?

**Ответ:** Нет, не нужно. ManagerFix совместим с большинством плагинов. Однако, если у вас уже установлены плагины с аналогичным функционалом (EssentialsX, AdvancedTeleport, и т.д.), могут возникнуть конфликты команд. В этом случае:
- Отключите дублирующие команды в `plugin.yml` другого плагина
- Или отключите соответствующие модули в `config.yml` ManagerFix

### ❓ Как обновить плагин?

**Ответ:**
1. Сохраните папки `config.yml`, `modules/`, `lang/` в безопасное место
2. Остановите сервер
3. Удалите старый `.jar` файл из `plugins/`
4. Поместите новый `.jar` файл
5. Запустите сервер
6. При необходимости обновите конфиги, сравнив с новыми версиями

### ❓ Теряются ли данные при обновлении?

**Ответ:** Нет, данные не теряются. Все данные хранятся в отдельных файлах в папке `data/` или в базе данных. При обновлении сохраняется:
- Профили игроков
- Варпы и дома
- Баны и муты
- Киты
- Предметы

### ❓ Можно ли использовать ManagerFix на Sponge/Forge?

**Ответ:** Нет, ManagerFix разработан только для **Paper/Spigot** серверов версии 1.21.x. Для Sponge существует отдельная версия (если доступна).

### ❓ Поддерживается ли BungeeCord/Velocity?

**Ответ:** Нет, ManagerFix работает только на отдельных серверах. Для сети серверов можно настроить кластер Redis для синхронизации данных между серверами.

## ⚙️ Конфигурация

### ❓ Как включить/отключить модуль?

**Ответ:** Откройте `config.yml` и измените значение нужного модуля:

```yaml
modules:
  warps: true      # Включено
  homes: false     # Отключено
  chat: true
```

### ❓ Как изменить сообщения плагина?

**Ответ:** Откройте файл `plugins/ManagerFix/lang/ru.yml` и измените нужные сообщения. Все сообщения используют формат **MiniMessage**:

```yaml
messages:
  no-permission: "<red>У вас нет прав!</red>"
  cooldown: "<red>Подождите ещё <yellow>{seconds}</yellow> сек.</red>"
```

### ❓ Как настроить форматы чата?

**Ответ:** В файле `modules/chat/config.yml`:

```yaml
format-local: "{badge} {prefix}{player}{suffix}: {message}"
format-global: "{badge} {prefix}{player}{suffix}: {message}"
badge-local: "｢𝐋｣"
badge-global: "｢𝐆｣"
```

### ❓ Как изменить кулдауны команд?

**Ответ:** В файле `modules/<модуль>/commands.yml`:

```yaml
commands:
  warp:
    enabled: true
    cooldown: 5              # Кулдаун 5 секунд
    bypass-permission: managerfix.warps.bypass.cooldown
```

### ❓ Как сменить язык плагина?

**Ответ:** В `config.yml`:

```yaml
settings:
  default-language: ru       # ru, en, etc.
```

Языковые файлы находятся в `plugins/ManagerFix/lang/`

### ❓ Как настроить лимиты домов?

**Ответ:** Лимиты задаются правами:

```
managerfix.homes.limit.1    — 1 дом
managerfix.homes.limit.3    — 3 дома
managerfix.homes.limit.5    — 5 домов
managerfix.homes.limit.10   — 10 домов
```

Назначьте права игроку через плагин прав (LuckPerms, PermissionsEx, и т.д.)

### ❓ Как изменить иконки в GUI?

**Ответ:** В конфиге модуля (например, `modules/warps/config.yml`):

```yaml
gui:
  navigation:
    previous-page:
      slot: 48
      icon: ARROW            # Название материала
      name: "&cНазад"
```

Список материалов: https://hub.spigotmc.org/javadocs/spigot/org/bukkit/Material.html

### ❓ Как настроить авто-сохранение профилей?

**Ответ:** В `config.yml`:

```yaml
settings:
  profile-autosave-minutes: 10    # Сохранять каждые 10 минут
```

## 💾 База данных

### ❓ Как переключиться на MySQL?

**Ответ:** В `config.yml`:

```yaml
storage:
  type: MYSQL              # Было YAML

database:
  host: localhost
  port: 3306
  database: managerfix
  username: root
  password: your_password
  pool-size: 10
```

Перезапустите сервер — данные автоматически мигрируют!

### ❓ Как создать базу данных?

**Ответ:** Выполните SQL запрос:

```sql
CREATE DATABASE managerfix CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

Или используйте готовый скрипт `setup_database.py`:

```bash
python setup_database.py
```

### ❓ Какие таблицы создаёт плагин?

**Ответ:**
- `mf_profiles` — профили игроков
- `mf_warps` — варпы
- `mf_kits` — киты
- `mf_bans` — баны
- `mf_mutes` — муты
- `mf_tpa_requests` — TPA запросы
- `mf_saved_items` — сохранённые предметы

### ❓ Как перенести данные с YAML на MySQL?

**Ответ:** Используйте команду миграции:

```
/migrate yaml2sql
```

Или просто измените `storage.type` на `MYSQL` в конфиге и перезапустите сервер.

### ❓ Как сделать резервную копию базы данных?

**Ответ:**

```bash
mysqldump -u root -p managerfix > backup.sql
```

Для восстановления:

```bash
mysql -u root -p managerfix < backup.sql
```

### ❓ Можно ли использовать MariaDB вместо MySQL?

**Ответ:** Да, MariaDB полностью совместима. Настройки в `config.yml` те же.

## 🧩 Модули

### ❓ Какие модули доступны?

**Ответ:**
- **Warps** — варпы с GUI
- **Homes** — дома игроков
- **Spawn** — спавн
- **Chat** — локальный/глобальный чат
- **TPA** — запросы телепортации
- **RTP** — случайная телепортация
- **Ban** — система банов
- **AFK** — режим «отошёл»
- **Kits** — наборы предметов
- **Worlds** — управление мирами
- **Other** — админ-утилиты
- **Tab** — кастомный таб-лист
- **Announcer** — объявления
- **Names** — кастомные ники
- **Items** — редактирование предметов

### ❓ Как отключить ненужные модули?

**Ответ:** В `config.yml`:

```yaml
modules:
  worlds: false            # Отключить миры
  announcer: false         # Отключить объявления
```

### ❓ Можно ли использовать только один модуль?

**Ответ:** Да, вы можете включить только нужные модули:

```yaml
modules:
  warps: true
  homes: true
  others: false            # Все остальные отключены
```

### ❓ Как настроить AFK кик?

**Ответ:** В `modules/afk/config.yml`:

```yaml
kick-timeout-seconds: 600    # Кик через 10 минут в AFK
```

### ❓ Как запретить создание домов?

**Ответ:** Отключите модуль Homes в `config.yml` или отберите права:

```
managerfix.homes.set       # Отобрать это право
```

### ❓ Как настроить кулдаун варпов?

**Ответ:** В `modules/warps/config.yml`:

```yaml
settings:
  cooldown: 5              # 5 секунд между телепортациями
```

### ❓ Как создать кит?

**Ответ:**
1. Возьмите предметы в руки
2. Выполните `/kit create <название>`
3. Или используйте GUI `/editkits`

### ❓ Как настроить RTP?

**Ответ:** В `modules/rtp/config.yml`:

```yaml
min-distance: 100          # Минимум 100 блоков от центра
max-distance: 5000         # Максимум 5000 блоков
cooldown: 30               # 30 секунд кулдаун
```

## 🎫 Команды и права

### ❓ Как выдать права игроку?

**Ответ:** Используйте плагин прав (LuckPerms):

```
/lp user <игрок> permission set managerfix.warps.use
/lp user <игрок> permission set managerfix.homes.limit.5
```

### ❓ Какие права у операторов?

**Ответ:** По умолчанию операторы (OP) имеют все права плагина.

### ❓ Как дать доступ к конкретному варпу?

**Ответ:**

```
/lp user <игрок> permission set managerfix.warps.warp.spawn
```

### ❓ Как запретить игроку использовать команды?

**Ответ:** Отоберите право `managerfix.command.<команда>`:

```
/lp user <игрок> permission set managerfix.command.rtp false
```

### ❓ Что такое bypass права?

**Ответ:** Права для обхода ограничений:

```
managerfix.bypass.cooldown     # Обход кулдаунов
managerfix.bypass.limit          # Обход лимитов
```

### ❓ Как посмотреть список всех прав?

**Ответ:** См. файл `plugin.yml` или документацию [COMMANDS.md](COMMANDS.md)

## ⚠️ Проблемы и ошибки

### ❓ Плагин не включается

**Ответ:** Проверьте:
1. Версию Java (требуется Java 21)
2. Версию сервера (требуется Paper 1.21.x)
3. Логи консоли на наличие ошибок

### ❓ Команды не работают

**Ответ:** Проверьте:
1. Права доступа у игрока
2. Включён ли соответствующий модуль в `config.yml`
3. Нет ли конфликтов с другими плагинами

### ❓ GUI не открывается

**Ответ:** Проверьте:
1. Включён ли модуль
2. Нет ли ошибок в консоли
3. Версию ProtocolLib (если используется)

### ❓ База данных не подключается

**Ответ:** Проверьте:
1. Правильность данных в `config.yml`
2. Доступность MySQL сервера
3. Права пользователя БД
4. Создана ли база данных

### ❓ Данные не сохраняются

**Ответ:** Проверьте:
1. Права на запись в папку `plugins/ManagerFix/`
2. Достаточно ли места на диске
3. Нет ли ошибок в консоли

### ❓ Игроки жалуются на лаги

**Ответ:** Попробуйте:
1. Увеличить интервал автосохранения
2. Отключить ненужные модули
3. Переключиться на MySQL (если много игроков)

### ❓ Миграция данных не работает

**Ответ:** Проверьте:
1. Есть ли данные в исходном хранилище
2. Подключена ли база данных
3. Логи консоли на наличие ошибок

### ❓ PlaceholderAPI не работает

**Ответ:** Проверьте:
1. Установлен ли PlaceholderAPI
2. Зарегистрирована ли экспансия (должно быть в логах)
3. Правильность использования плейсхолдеров (`%managerfix_afk%`)

### ❓ Vault не интегрируется

**Ответ:** Проверьте:
1. Установлен ли Vault
2. Установлен ли плагин экономики
3. Логи на наличие ошибок интеграции

## 🔌 Интеграция

### ❓ Совместим ли плагин с LuckPerms?

**Ответ:** Да, ManagerFix полностью совместим с LuckPerms и использует его для проверки прав.

### ❓ Работает ли плагин с Vault?

**Ответ:** Да, Vault используется для:
- Отображения баланса в тултипах чата
- Префиксов/суффиксов в форматах чата

### ❓ Можно ли интегрировать с Discord?

**Ответ:** Да, через API плагина. См. [DEVELOPER_GUIDE.md](DEVELOPER_GUIDE.md)

### ❓ Работает ли плагин с ProtocolLib?

**Ответ:** Да, ProtocolLib используется для некоторых функций GUI и тултипов.

### ❓ Можно ли использовать с другими плагинами телепортации?

**Ответ:** Да, но могут быть конфликты команд. Отключите дублирующие команды в другом плагине или в ManagerFix.

## ⚡ Производительность

### ❓ Влияет ли плагин на TPS?

**Ответ:** Минимально. Плагин использует асинхронные операции для:
- Работы с базой данных
- Сохранения профилей
- Загрузки данных

### ❓ Сколько памяти потребляет плагин?

**Ответ:** В среднем 50-100 MB в зависимости от количества модулей и игроков.

### ❓ Как оптимизировать производительность?

**Ответ:**
1. Используйте MySQL вместо YAML при большом количестве игроков
2. Отключите ненужные модули
3. Увеличьте интервал автосохранения профилей
4. Уменьшите частоту обновления таб-листа

### ❓ Сколько игроков поддерживает плагин?

**Ответ:** Плагин протестирован на серверах с 100+ игроками. При правильной настройке (MySQL, оптимизация) поддерживает неограниченное количество игроков.
